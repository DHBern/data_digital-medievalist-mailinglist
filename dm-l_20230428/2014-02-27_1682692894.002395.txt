Subject: [dm-l] xslt functions for roman dates and numeralsDear allI've written two XSL 2.0 functions which might be of some interest for =20
digital medievalists.
The first ("is_roman_number") checks if a given string is a correct =20
Roman numeral or not (returning a boolean).
E.g. <xsl:value-of select=3D"locfun:is_roman_number('xvi')"/> returns "true".
The second ("convert_date") converts Roman dates (you know, something =20
like 'XII Kalendas Decembris') into a string, containing the date in =20
modern (english) format.The function expects four parameters:
1. A string indicating the month in latin. The function is quite =20
tolerant here.
2. A string indicating the part of the month: kalendae, nonae, idus. =20
Again much spelling variation tolerance here.
3. A string indicating the given number of the day in Roman numerals. =20
If your date is exactly on a fix date (eg. "kalendibus"), just write =20
'' here.
4. A boolean for the leap year. 'true()' means 'yes, we're in a leap year'.E.g. <xsl:value-of =20
select=3D"locfun:convert_date('iunii','idus','III',false())"/> will =20
return "June 11".Both functions use the xmlns:locfun=3D"urn:local-functions" namespace =20
and the =20
xmlns:own_t=3D"http://no-real-uri-here/arguments_and_document_node_variables_=
of_Christian_Schwaderer" namespace for document node =20
variables.You find the code file attached. You might use, edit, share it in =20
whichever way you like!Best regards
Christian
